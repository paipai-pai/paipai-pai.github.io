SQL注入原理

搭建好sqli-labs靶场，当我们访问?id=1的时候就是去数据库里查询id=1的字段对应的值，同理id=2，id=3也是一样。一般情况下我们进行简单的注入识别。用到?id=1’，如果存在注入点，这里就会发生报错。

看到报错回显：你的SQL语法有误；请检查你的mysql服务器对应的手册以获取正确的语法，以在第一行的’’1’’LIMIT 0,1’附近使用。分析一下单引号的配对方式：
也就是说输入1‘时，单引号提前闭合导致了问题的出现，less1的源代码如下：

这里我们可以看到查询语句：SELECT * FROM users WHERE id=’$id’ LIMIT 0,1 。假如这里的$id的只为我们输入的1’，传入的引号就会把原本的引号提前闭合，多出来的的单引号就导致了语法错误。

SQL注入流程

1 寻找SQL注入点

目标搜集：
  无特定目标：
    inurl:.php?id=
  有特定目标：
   inurl:.php?id= site:target.com
  工具爬取
   spider，对搜索引擎和目标网站的链接进行爬取
注入识别
  手工简单识别：
   and1=1/and1=2
   and’1’=1’1/and’1’=2’
   and1like1/and1like2
  工具识别：
   sqlmap -m filename（filename中保存检测目标）
   sqlmap –crawl（sqlmap对目标网站进行爬去，然后依次进行测试）
高级识别
  扩展识别广度和深度：
   Sqlmap –level 增加测试级别，对header中相关参数也进行测试
   sqlmap -r filename（filename中为网站请求数据）
  利用工具提高识别效率：
   BurpSuite + SqlMap
   BurpSuite拦截所有浏览器访问提交的数据
   BurpSuite扩展插件，直接调用SqlMap进行测试
 一些Tips：
   可以在参数后键入”*”来确定想要测试的参数
   可能出现的注入点:新闻、登录、搜索、留言…
   站在开发的角度去寻找
代码审计
 搜索关键代码和函数
 梳理业务流程

2 MySQL手工注入方法

MySQL数据库结构
  存储引擎层：主要实现数据存取，定义数据存储的格式
  SQL层： 解析器、授权、优化器、查询执行、查询告诉缓存、查询日志记录
  连接层：通信协议、线程、验证
MySQL手工注入方法
|函数名称 | 函数功能|
| —- | —- |
|查库 | select schema_name from information_schema.schemata|
|查表 | select table_name from information_schema.tables where table_schema=库名|
|查列 | select column_name from information_schema.columns where table_name=表名|
|查数据 | select列名from库名.表名|
