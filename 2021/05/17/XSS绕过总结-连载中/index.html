HTML-源代码注入

当输入作为以下HTML标记属性的值时使用：href,src,data或action（也是格式）。脚本标记中的Src属性可以是URL或“data:alert(1)”

JavaScript-代码注入

当输入落在脚本块中时，在字符串分隔值内使用。

JavaScript-使用Escape Bypass进行代码注入

当输入落在脚本块中时，在字符串分隔值内使用，但引号由反斜杠转义。

JavaScript-标记注入

当输入落在脚本块中的任何位置时使用。

0x02 进阶注入方式

JavaScript-逻辑块中的代码注入

当输入落在脚本块中时，使用第一个或第二个有效负载，在字符串分隔值内，并在单个逻辑块内，如函数或条件(if,else,etc)。如果使用反斜杠转义quote，请使用第三个有效负责块。

JavaScript-无引用代码注入

在同一行JS代码中存在多重反射时使用。第一个有效负载在简单的JS变量中工作，第二个在非嵌套的JS对象中工作。

JavaScript-模板文字中的占位符注入

当输入在反引号(”)分隔的字符串或模板引擎中时使用。

HTML-多重反射-双反射（单输入）

用于利用同一页面上的多个反射。

HTML-多重反射-三重反射（单输入）

用于利用同一页面上的多个反射。

HTML-多输入反射（双重和三重）

用于在同一页面上利用多个输入反射。在HPP（HTTP参数污染）场景中也很有用，其中存在重复参数的反射。第3个有效负载使用相同参数的逗号分隔反射。

文件上传注入-文件名

当上传的文件名反应在目标页面的某处时使用。

文件上传注入-元数据Metadata

在上传文件的元数据反应在目标页面的某处时使用，它使用命令行exiftool（“$”是终端提示符），可以设置任何元数据字段。

文件上传注入-SVG文件

用于在上载图像文件时在目标上创建存储的XSS。将以下内容保存为
“xss.svg”.

DOM插入注入

当注入作为有效标记插入DOM而不是反映在源代码中时，用于测试xss。他适用于脚本标记和其他向量不起作用的情况。

DOM插入注入-资源请求Resource Request

当页面的JavaScript代码将请求的结果插入页面时，使用对攻击者控制的URL（注入）。

PHP_SELF注入

例如。当目标的底层PHP代码将URL用作HTML表单的属性值时使用。使用前导斜杠（/）在PHP扩展和查询部分（？）的开始之间进行注入。

脚本注入-没有关闭标记

在反射后代码中的某处有一个关闭脚本标记时使用。

JavaScript postMessage() DOM注入（使用iframe）

在没有检查原点的JavaScript代码中的”windows.addEventListener(‘message’,…)”中
